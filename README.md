# gosso

gosso is a web project generated by gouno.

> gouno is a Go language web application scaffold designed to provide a fast and reliable starting point for building production-grade web services. It integrates the Gin framework, Cobra CLI tools, and Viper configuration management, and follows a clear architectural design to help developers focus on business logic implementation.

## Features

-   **Gin Framework**: A high-performance HTTP web framework.
-   **Cobra CLI**: A powerful command-line interface tool for building CLI applications.
-   **Viper Configuration Management**: A flexible configuration solution supporting multiple formats and hot reloading.
-   **Clear Architecture**: Adopts a layered design for easy understanding and extension.
-   **Graceful Shutdown**: Supports graceful shutdown to ensure requests are properly handled.
-   **Project Structure**: Follows standard Go project layout, including cmd, internal, config, and other directories.
-   **Makefile Support**: Provides shortcuts for common commands to streamline the development process.

## Installation and Running

### Prerequisites

-   Go 1.23.3+ environment

### Clone Project

```bash
git clone gosso
cd gosso
```

### Build Project

Use the `make` command:

```bash
make
```

### Run Web Service

```bash
./bin/gouno web --config ./config/config.yaml --address 0.0.0.0 --port 8080
```

By default, the web service will start at `http://0.0.0.0:8080`.

#### Usage Instructions

##### Command Line Arguments

-   `--config` or `-c`: Specifies the configuration file path, defaults to `./config/config.yaml`.
-   `--address` or `-a`: Specifies the listening address, defaults to `0.0.0.0`.
-   `--port` or `-p`: Specifies the listening port, defaults to `8080`.
-   `--debug` or `-d`: Enables debug mode, defaults to `false`.

### Development Mode

```bash
make dev
```
